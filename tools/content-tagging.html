Edvance · Content-QS & Tagging

Hilfe / Anleitung
  • Internes Tool — kein Produkt. Verbindet sich mit Supabase und taggt die screening_items-Itembank.
  • URL + anon-Key aus deiner .env (VITE_SUPABASE_URL / VITE_SUPABASE_ANON_KEY). Werden nur lokal im Browser (localStorage) gespeichert, nie im Repo.
  • Login mit Admin-E-Mail/Passwort — nur admin darf schreiben (RLS screening_items_admin_all). Passwort wird nicht gespeichert.
  • Pro Item: Cluster (Achse A), Kompetenz (Achse B), Microskill (feinste Granularität, 043) setzen → Speichern.
  • aktiv-Schalter macht ein Item im Screening für Schüler:innen sichtbar — mit Bestätigung.
  • QS-Badges zeigen automatische Qualitäts-Probleme (fehlende Felder, Cluster-Mismatch, unvollständig & aktiv …).
